# Newsmemo
**ニュースを能動的に読むことに特化したアプリです。**

**URL** : https://newsmemo.site/login


## アプリ概要
### アプリケーションのコンセプト
アプリケーションのコンセプトは、以下の2点です。

1.読み終えたニュースや気になるニュースをメモ(投稿)することで、ニュースを読む習慣化を図る
<br>
2.ニュースに対するマインドマップを作成することで、考える力を習得する
### アプリケーションの特徴
基本的には、某SNSのような投稿、いいね、フォロー機能があるSNSですが、その他に以下のような特徴があります。

* 基本的に他のユーザーと直接関わるような機能はあえて実装していない<br>
(いいね機能は、ニュースに対する他のユーザーの意見や、見落としたニュースを記録するために実装)<br>
(フォロー機能は、自分とは違った視点からニュースを見ているユーザーなど、気になったユーザーを見失わないために実装)

* アプリ内で、ニュースの情報を得ることができる(NewsAPI連携)
* 投稿累計日数やメモ総数を記録することができる
* ユーザーに人気なニュースや投稿のランキング機能
* 投稿(メモ)に対してタグ機能が使用可能であり、カテゴリ別にニュースに関する意見等を共有できる
* 投稿(メモ)に対してコレクション機能が使用可能であり、各ユーザーがジャンル別にニュースを管理できる(非公開)
* 投稿(メモ)に対して非公開コメント機能(マインドマップ機能)が使用可能であり、ニュースに対するマインドマップを作成できる
* マインドマップ機能を使うことで、考える力を養うことができる
* 読んだニュースをTwitterで共有することができる

## 開発した背景
どのような需要を満たすサービスを作成するか悩んだ上で、まずは初めの一歩として自分の需要を満たすサービスを作ろうと思い、今回の開発に至りました。<br>
<br>
学生の頃はニュースを受動的に見ていたが、
<br>
社会人になったタイミングで能動的にニュースを読むことを習慣化したいと思ったことがきっかけです。<br>
また、周囲の友人もニュースを読むことが習慣化できていなかったので、少しでも役立てればと思いこのサービスを考えました。
<br>
<br>
アプリケーションを作成する前は、<br>
既存のアプリケーションを使い能動的にニュースを読むことを実践しましたが、主に以下のような問題が原因で習慣化できずにいました。

* アプリケーション上のコメント欄でユーザー同士の言い争いが目に入り不快である
* 各ユーザーが個別でニュースを管理できる機能が少ない
* 習慣化に繋がる機能が実装されていない

上記のような課題を解決したアプリケーションを作成することによって、結果的にニュースを能動的に読むことを習慣化できると考えたことが開発した背景にあります。

今後は、様々な市場のニーズを満たせるようなサービスを生み出したいため、常にアンテナを張り巡らせ、ビジネスチャンスを逃さないようにしたいと考えています。

## 機能一覧
* ユーザー登録関連機能
  * アカウント新規登録、プロフィール編集機能、退会機能（論理削除）
  * パスワード変更機能
  * ログイン、ログアウト機能
  * ゲストユーザーログイン機能(ユーザー登録なし)
  * パスワードリセット機能(現在利用停止)

* NewsAPI連携
  * NewsAPIを利用したニュース情報の取得

* ニュースと投稿に関するランキング機能（1ヶ月毎）
  * １ヶ月ごとによく読まれているニュースの記事をランキング形式で表示
  * １ヶ月ごとにユーザーから共感された(いいね)数が多い投稿をランキング形式で表示

* 画像アップロード機能
  * プロフィールのアイコンとして利用する画像を保存できます(AWS S3バケットを利用)

* コレクション機能
  *　各ユーザーが投稿をカテゴリ別に管理できる(他ユーザーの投稿も登録・削除可能)
  *　コレクションを保存する際は非同期処理で実行(Vue.jsを使用)

* ページネーション機能
  * 投稿一覧、コレクション機能での投稿一覧など

* ユーザー投稿機能(CRUD)
  * CRUD機能
  * クイック投稿機能・・・「クイックメモ」をクリックすると、「後で読む。」というテンプレート文を投稿できます。とりあえず投稿したい時に使用します

* マインドマップ機能
  * ニュースに関する気づきや考えを投稿の詳細ページで記録できます。(非公開の状態なため、他のユーザーには見られません)
  * 編集・削除可能

* タグ機能
  * タグごとに投稿を一覧表示できます

* フォロー機能
  * フォロー/フォロワーの一覧ページでユーザーの確認ができます
  * フォローボタンのアクションは非同期処理で実装(Vue.jsを使用)

* いいね機能(ブックマーク機能)
  * 他のユーザーの投稿内容に共感した際にご利用ください。
  * 投稿一覧画面で気になるニュースがあれば、いいね機能を利用して後からでもすぐに読めるよう記録できます。
  * いいねの登録は非同期処理で実行します(Vue.jsを使用)

* 検索機能
  * 投稿の検索

* フラッシュメッセージ表示機能
  * 投稿、編集、削除、ログイン、ログアウト時にフラッシュメッセージを表示する

* レスポンシブデザイン
  * モバイルファーストをベースとしたデザイン

### 各テーブルについて

|テーブル名|説明|
|:--|:--|
|users|一般ユーザー情報|
|follows|フォロー中/フォロワーのユーザー情報|
|contacts|お問い合わせに関する情報|
|articles|ユーザー投稿の情報|
|tags|ユーザー投稿のタグ情報|
|article_tag|articlesとtagsの中間テーブル|
|collections|ユーザー投稿のコレクション情報|
|article_collection|articlesとcollectionsの中間テーブル|
|likes|投稿への「いいね(ブックマーク)」に関する情報|
|news_links|ユーザー投稿のニュース情報|
|memos|ユーザー投稿のマインドマップ情報|
